Two Dimensions

Simulate trajectories of a particle cloud in a two-dimensional flow field. A doubly-periodic domain and simple flow fields are used. For additional documentation e.g. see : 1, 2, 3, 4

Exercises:

  • change the initial distribution of particles
  • increase the duration of the trajectories simulation
  • treat the non-periodic domain case by padding u,v with zeros
  • replace u,v with your own two-dimensional flow fields

2. Flow Fields

Let's start with a simple, doubly periodic flow field defined by a streamfunction and create the FlowFields data structure which will then be drive the individual displacement and trajectory computations

nx=16

XC = dx*(collect(1:2*nx) .- 0.5)

fac=0.1
f(x, y) = sin(x) + cos(y) #streamfunction
Ï• = fac*[f(x, y) for x in XC,y in YC] #streamfunction
uC = -fac*[sin(y) for x in XC,y in YC] #dphi/dy at cell center
vC = -fac*[cos(x) for x in XC,y in YC]; #-dphi/dx at cell center

When starting as done here with collocated velocities in m/s units (uC,vC at the grid cell center), one easily obtains the normalized, staggered C-grid velocities (u,v) as shown below. u (resp v) is staggered by -0.5 grid point in direction 1 for u (2 for v) from the grid cell center uC,vC. This staggering of variables as well as the normalization of velocities by the grid scale are key conventions adopted in IndividualDisplacements.jl.

u=0.5*(circshift(uC, (1,0))+uC) /dx #staggered u converted to grid point units (m/s -> 1/s)
v=0.5*(circshift(vC, (0,1))+vC) /dx #staggered v converted to grid point units (m/s -> 1/s)

6. Alternatives (optional)

Making the flow field time variable simply involves providing two different flow fields. Linear interpolation will then be applied in the specified time range. For example :

ğ¹=FlowFields(-u,u,-v,v,[0.,10.])

It is often convenient to set up FlowFields using the MeshArrays.jl package, which handles such staggered grids in general fashion, instead of plain Arrays. Given a doubly periodic flow field defined as before this is readily done via the convert_to_FlowFields function.

u,v,Ï•=random_flow_field()
#ğ¹=FlowFields(u,u,v,v,[0.,10.])
ğ¹=convert_to_FlowFields(u,v,10.0);

We should note that u,v derive from streamfunction Ï• which is defined at the corner point in random_flow_field. Doing this ensures that the resulting u,v is non-divergent / purely-rotational over the C-grid domain. In brief:

u=-(circshift(Ï•, (0,-1))-Ï•)
v=(circshift(Ï•, (-1,0))-Ï•)

In contrast uC, vC computed as earlier in this example may contain contain both rotational and divergent components â€“ if this were an important consideration, MeshArrays.jl provides tools that can be used to e.g. ensure non-divergence.

When using MeshArray flow fields instead of plain Array flow fields, initial positions also include a subdomain array index (a; all ones in our example).

a=ones(size(x))
#isa(ğ¹,ğ¹_MeshArray2D)
ğ¼=Individuals(ğ¹,x,y,a)
